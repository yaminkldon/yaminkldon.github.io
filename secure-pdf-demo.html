🔒 Secure PDF Viewer Demo

Security Features

  • URL Hiding - PDF URLs are obfuscated and not exposed in browser tools
  • Print Prevention - All printing methods are blocked
  • Copy Prevention - Text selection and copying is disabled
  • Screenshot Protection - Print Screen detection and clipboard clearing
  • Right-click Disabled - Context menu is completely blocked
  • Developer Tools Detection - Automatically closes when dev tools are opened
  • Keyboard Shortcuts Blocked - F12, Ctrl+P, Ctrl+S, Ctrl+C, etc.
  • Watermarking - User email and timestamp overlays
  • Session Tracking - Monitors viewing time and user activity
  • Secure Loading - Custom loading screen with security indicators

How to Use

1. Basic Usage:

<!-- Direct link to secure PDF viewer --> <a href="secure-pdf-viewer.html?file=path/to/your/document.pdf&user=student@example.com"> Open Secure PDF </a>

2. JavaScript Integration:

// Function to open secure PDF viewer function openSecurePDF(pdfUrl, userEmail) { const viewerUrl = 'secure-pdf-viewer.html?' + 'file=' + encodeURIComponent(pdfUrl) + '&user=' + encodeURIComponent(userEmail) + '×tamp=' + Date.now(); window.open(viewerUrl, '_blank', 'width=1200,height=800'); } // Example usage openSecurePDF('documents/lesson1.pdf', 'john.doe@school.edu');

3. Embedded in Modal:

// Create modal with embedded secure PDF viewer function showPDFModal(pdfUrl, userEmail) { const modal = document.createElement('div'); modal.style.cssText = ` position: fixed; top: 0; left: 0; width: 100%; height: 100%; background: rgba(0,0,0,0.8); z-index: 10000; display: flex; justify-content: center; align-items: center; `; const viewerUrl = 'secure-pdf-viewer.html?' + 'file=' + encodeURIComponent(pdfUrl) + '&user=' + encodeURIComponent(userEmail); modal.innerHTML = ` <iframe src="${viewerUrl}" style="width: 90%; height: 90%; border: none; border-radius: 8px;"> </iframe> `; document.body.appendChild(modal); }
Security Note: This secure PDF viewer is designed for educational platforms where content protection is important. It implements multiple layers of security but should be used as part of a comprehensive security strategy that includes server-side access controls and proper user authentication.
